# 线框图开发

这项任务应该以你自己的评估项目为目标。

**阅读材料**
- 什么是线框图？ - [https://balsamiq.com/learn/articles/what-are-wireframes/](https://balsamiq.com/learn/articles/what-are-wireframes/)
- 如何绘制线框图？ - [https://www.figma.com/blog/how-to-wireframe/](https://www.figma.com/blog/how-to-wireframe/)

##### **任务**
- 从你的流程日志中取出一张 A4 纸
- 确定一个你想绘制线框图的活动，可以是应用程序、网站或实体界面
- 列出用户完成该活动目标所需的步骤，例如，如果你要登录邮箱，你需要输入用户名、密码并点击一个按钮。
- 确定每个步骤所需的元素。问问自己，同一屏幕上应该显示多少信息？
- 绘制该步骤活动所需的界面元素，例如，登录屏幕需要两个文本框和一个按钮。
- 重复此步骤，直到您为您的设计方案创建了一个涵盖您计划规划的整个活动的线框图。
- 使用 Figma 或类似的图形软件重新创建您绘制的线框图。如果您的线框图是用于应用程序或网站，请考虑添加交互元素，例如点击跳转到其他屏幕或滚动等。
- 观察线框图，识别用户在您的界面中可以采取的多种路径和交互方式，以实现预期目标。参考工作簿中活动的启发式方法和经验教训来评估您的线框图。
- **提交：** 将线框图添加到您的流程日志中，或按说明提交到论坛。尽量清晰地描述交互方式，尤其是在界面之间切换时，例如在智能手机和智能手表之间切换等。
