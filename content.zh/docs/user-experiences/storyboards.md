# 故事板

本次活动的目的是探索故事板的创作风格。查找人们制作故事板的各种方法，并制作一个风格参考板。找到一种适合你技能和兴趣的故事板风格。记住，故事板不应该只是一系列屏幕或一些文字，它应该描绘用户及其所处的环境。故事板可以用来理解用户体验中的问题，也可以用来规划用户体验的流程。为了完成这项任务，我们假设你已经构思了一个可能的设计方向，并将在故事板中体现出来。第六周提供了一个故事板模板。

- **阅读材料：**《交互设计》- Preece，第 426-428 页，12.2.3 低保真原型

## 任务

- 将你构思的用户体验分解成关键步骤，并按操作顺序写出来。
- 取一张 A4 或 A3 纸，画出若干个方框，用来展示故事的场景。每个方框下方留出一些空间，以便描述场景。确保有足够的空间绘制场景本身，不要把故事板画得太小！
- 将任务开始时确定的每个步骤都画成一个单独的场景。可以先在方框下方描述场景，然后再画出来。
- 继续绘制，直到所有步骤都用单独的方框画满整张纸。
	- 你的故事板是否包含了以下内容？
	- 故事板是否展示了用户体验的开始和结束？
	- 故事板是否展示了用户体验可能失败的情况？
	- 故事板是否展示了用户体验可能成功的情况？

- 如果以上任何问题的答案是“否”，请考虑在故事板中添加或删除相应的步骤。
