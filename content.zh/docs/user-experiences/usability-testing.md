# 可用性测试

可用性是指某事物的易用性。在交互设计领域，可用性指的是界面的易用性。测试设计的可用性至关重要，它可以确保设计符合用户的期望，并解决设计过程中定义的问题。本次活动的目的是了解您的原型在用户使用中的效果。

**注意：** 请谨慎选择测试对象，务必始终遵循用户研究和测试的伦理规范。**切勿让儿童测试您的设计。测试对象应仅限于您的朋友和家人。**

- **阅读材料：**《交互设计》- Preece，第 10 页496-509
- 14.2 评估的目的、内容、地点和时间
- 14.3 评估类型

## 任务

- 对您最新的设计进行可用性测试，并注意以下几个方面。测试对象为五位用户，以全面了解出现的问题。
	- 完成一项任务所需时间。
	- 用户离开产品一段时间后完成另一项任务所需时间。
	- 每项任务的错误数量和类型。
	- 单位时间内出现的错误数量。
	- 访问在线帮助或手册的次数。
	- 出现特定错误的用户数量。
	- 成功完成任务的用户数量。
- 将您的发现整理成一份可用性报告，请参考 Moodle 上提供的指南。
