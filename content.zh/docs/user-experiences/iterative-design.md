# 迭代设计

本次活动的目的是回顾前几周探讨的概念，并鼓励您对界面可用性进行细微的改进。您可能需要重新思考工作簿中各项任务提出的一些问题。

- **阅读材料：** 交互设计基金会 - 设计迭代带来强大的成果，设计师，再来一次！[https://www.interaction-design.org/literature/article/design-iteration-brings-powerful-results-so-do-it-again-designer](https://www.interaction-design.org/literature/article/design-iteration-brings-powerful-results-so-do-it-again-designer)

## 任务

- 回顾一些关于您目前设计的核心问题：
	- 用户是否可以通过探索界面和尝试某些操作来弄清楚如何使用产品？
	- 用户学会如何使用该设计完成任务后，能否保持较高的效率（例如，是否需要其他资源，能否随时随地完成任务）？
	- 该设计的功能是否足以支持用户完成所有任务？有哪些与用户需求或问题相关的任务无法通过该设计完成？
	- 与其他完成相同任务的方法相比，该界面是否更便捷？

	- 该界面是否可以根据特定需求或无障碍要求进行修改？
- 重新审视您的用户画像。
	- 不同用户对该设计的使用方式有何不同？
	- 该设计是否更偏向于某一类用户？
	- 是否有多种方法可以实现同一目标？
	- 如果遇到截然不同的用户类型，会发生什么情况？
